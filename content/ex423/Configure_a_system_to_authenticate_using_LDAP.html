--- title: Configure a system to authenticate using LDAP objective: 18 category: EX423 --- Once the server is setup this part is easy. Just install clients, configure system, tweak pam. #### Install openldap-clients on client [root@client01 ~]# yum -y install openldap-clients #### Install the cacert from your CA. /etc/openldap/cacert.crt ####3 Configure the ldap url and base dn to match what you used edit /etc/openldap/ldap.conf URI ldap://dc03.makewhatis.com ldap://dc03.makewhatis.com:636 BASE dc=makewhatis, dc=com TLS_CACERT /etc/openldap/cacert.crt Now you should be able to run [root@client01 ~]# ldapsearch -x -ZZ ### Configure it to authenticate This is a nice one liner. You could also hit system-config-authentication to do this, or system-config-authentication-tui, but I like just using the cmd tools. [root@client01 ~]# authconfig --enableldap --enableldapauth \ --disablenis --enablecache --ldapserver=dc03.makewhatis.com \ --ldapbasedn=dc=makewhatis,dc=com --updateall If this is a system that you want home directories auto created, then you need to add the pam module for that to system-auth. An easy way to find it is with man -k [root@client01 ~]# man -k pam_ | grep home pam_mkhomedir (8) - PAM module to create users home directory On `pam_mkhomedir` man page you can see instructions on usage. [Red Hat Docs on PAM](http://docs.redhat.com/docs/en-US/Red_Hat_Enterprise_Linux/5/html-single/Deployment_Guide/index.html#ch-pam)
